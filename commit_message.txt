fix(gateway): develop 브랜치 병합 후 발생한 인증 오류 해결

develop 브랜치와 병합하는 과정에서 발생한 인증 문제를 해결했습니다.

문제 원인:
API 게이트웨이(gateway-service)의 AuthorizationHeaderFilter에 /attendance/** 경로가 허용 목록(allowUrl)에 잘못 포함되어 있었습니다. 이로 인해 attendance-service로 향하는 모든 요청이 인증 검사를 거치지 않고 통과되어, 후속 서비스에서 사용자 정보(Email, Role 등)를 받지 못하는 오류가 발생했습니다.

해결 내용:
AuthorizationHeaderFilter의 allowUrl 리스트에서 /attendance/** 항목을 제거했습니다. 이로써 게이트웨이는 attendance-service로 들어오는 모든 요청에 대해 정상적으로 JWT 토큰을 검증하고, 사용자 정보를 담은 헤더(X-User-Email 등)를 추가하게 됩니다.

기대 효과:
이번 수정을 통해 attendance-service가 게이트웨이를 통해 안전하게 인증된 요청만을 처리하게 되어 시스템 전반의 보안 및 안정성이 향상되었습니다.
